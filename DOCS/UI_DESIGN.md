# DUCKAVERB UI Design Guide

## 디자인 철학

DUCKAVERB의 UI는 **기타 페달**에서 영감을 받아, 단순하면서도 직관적인 사용자 경험을 제공합니다.

### 핵심 원칙

1. **미니멀리즘**: 불필요한 요소 제거, 본질에 집중
2. **명료성**: 모든 컨트롤의 기능이 즉시 이해 가능
3. **일관성**: FUZZA VST와 동일한 디자인 언어 사용
4. **반응성**: 실시간 피드백으로 사용자 입력에 즉각 반응

## 레이아웃

- **Width**: 380px

- 패널 테두리: `0x44ffffff` (27% 불투명도)
- 배경: 완전 불투명

## 컴포넌트 상세

### 로고 영역

```cpp
// 위치: 상단 70px
logoArea.removeFromTop(12); // 상단 패딩
g.setFont(juce::FontOptions(38.0f, juce::Font::bold));
g.drawFittedText("DUCKAVERB", ...);

// 서브타이틀
g.setFont(juce::FontOptions(13.0f, juce::Font::plain));
g.setColour(juce::Colour(0xffff9933));
g.drawFittedText("Ducking Reverb", ...);
```

### SPACE 노브 모듈

**박스 스타일**:
- **Border Radius**: 10px
- **Inset 효과**: 3px 검은색 섀도우
- **내부 패딩**: 15px (상하), 10px (좌우)

**노브 스타일**:
- **Type**: RotaryHorizontalVerticalDrag
- **Text Box**: 하단, 100px × 28px
- **Fill Color**: `#1e90ff` (파란색)
- **Outline**: `#333333` (어두운 회색)
- **Thumb**: 흰색 원형

**레이아웃**:

### BYPASS 버튼

**크기**: 150px × 45px

**상태별 색상**:
- **OFF**: 배경 `#333333`, 텍스트 `#eeeeee`
- **ON**: 배경 `#dd3333`, 텍스트 `#ffffff`

**위치**: 하단에서 80px, 중앙 정렬

### 장식 요소

#### 코너 나사

- **위치**: 각 코너에서 20px 안쪽
- **크기**: 12px 직경
- **스타일**:
  - 베이스: `#666666`
  - 하이라이트: `#aaaaaa`
  - 슬롯: `#333333`, 2px 두께 가로선

#### 테두리

- **외부 테두리**: 4px, `#000000`
- **내부 하이라이트**: 1px, `#444444`

## 반응형 동작

### 노브 인터랙션

1. **마우스 오버**: 변화 없음 (페달 리얼리즘 유지)
2. **드래그**: 상하 드래그로 값 변경
3. **더블 클릭**: 기본값(0.5)으로 리셋
4. **값 표시**: 실시간 업데이트 (0.00 형식)

### 버튼 인터랙션

1. **클릭**: 토글 상태 변경
2. **시각 피드백**: 색상 즉시 변경
3. **오디오 피드백**: 바이패스 즉시 적용

## FUZZA VST와의 일관성

DUCKAVERB는 FUZZA VST의 디자인 언어를 계승합니다:

### 공통 요소

- ✅ 다크 배경 (`#0f0f0f` ~ `#1f1f1f`)
- ✅ 오렌지 악센트 (`#ff9933`)
- ✅ 인셋 패널 스타일
- ✅ 중앙 정렬 BYPASS 버튼
- ✅ 코너 나사 디테일
- ✅ h4ppy Labs 푸터

### 차이점

| 요소 | FUZZA | DUCKAVERB |
|------|-------|-----------|
| 노브 수 | 3개 | 1개 |
| 버튼 수 | 6개 (토글) | 1개 (BYPASS) |
| 노브 색상 | 오렌지 | 파란색 |
| 레이아웃 | 3×3 그리드 | 단일 컬럼 |
| 서브타이틀 | "Modern Fuzz Pedal" | "Ducking Reverb" |

## 구현 참고사항

### paint() 함수 순서

1. 배경 그라데이션
2. 외부/내부 테두리
3. 로고 및 서브타이틀
4. 노브 박스 (배경 → 테두리)
5. 코너 나사
6. 푸터

### resized() 함수 순서

1. 로고 영역 할당
2. 푸터 영역 할당
3. BYPASS 버튼 영역 할당
4. 남은 공간을 노브 모듈에 할당
5. 노브 박스 중앙 정렬 (좌우 60px 여백)

## 접근성

- **색맹 고려**: 빨간색(BYPASS ON)과 파란색(노브) 명확히 구분
- **대비**: 모든 텍스트가 배경과 충분한 대비 (WCAG AA 이상)
- **크기**: 터치스크린 환경 고려한 최소 44px 버튼 크기
